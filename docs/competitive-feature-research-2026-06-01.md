# Byux 競合機能調査 — AviUtl / Adobe / DaVinci（2026-06-01）

## 0. 大前提（評価の軸）

Byux は **「FPSキル集を最短ルートで作る専門特化ツール」**。汎用NLE（Adobe / DaVinci / AviUtl）とは戦わない。
→ 競合の全機能をコピーするのは負け筋。**「キル集制作」に効く機能だけ**を抜き取り、**Byuxの簡単さ・速さを壊さない形（プリセット化／自動化）**で取り込む。

各機能は3軸で評価：
1. **速さ** — キル集制作のスピードを上げるか
2. **映え** — montage の見栄え（カッコよさ）を上げるか
3. **専門性維持** — 汎用化して簡単さを壊さないか

### Byux の現状機能（調査の基準）
キルマーカー自動分割 / I-Oレンジカット / BGM＋ビート検出＋ビート吸着 / フェード / 再生速度(¼×〜2×) / テキストオーバーレイ / モーションブラー(WebGL) / MP4書き出し(FFmpeg.wasm＋WebCodecs) / プロジェクト保存ロード / カスタムショートカット

## 4. 取り入れるべき機能（優先度付き・Byuxの"キル集を速くカッコよく"レンズで濾過）

   - キルの瞬間スロー→決め。**Byuxが既に持つモーションブラー資産を最も活かせる最有力機能**
3. **装飾テキスト＋テキストアニメ プリセット**（縁取り/グロー/登場演出）
   - 現状の素朴なテキストを montage 級へ。ワンクリック適用
4. **トランジション/シーンチェンジ プリセット**（キル間）

### 🥈 Tier 2 — 映像の質
5. **簡易カラーグレード/LUT**（"シネマ""鮮やか"等のワンクリックプリセット）
   - DaVinci発想を軽量実装（WebGLシェーダ/CSS filter）。**実装軽い割に見栄えのインパクト大**
6. **エフェクトプリセット**（グロー/シェイク/ズームパンチ）でキル強調
7. **オーディオ・ダッキング**（BGMをキル/SEに合わせ自動増減）×既存ビート検出

### 🥉 Tier 3 — 速度・快適さ

10. **プリセット/テンプレ保存**（自分の montage スタイルを使い回す → 共有/コミュニティ余地）

## 5. 横断的な気づき（insights）

1. **Byuxの勝ち筋は「キーフレーム ＋ プリセット」**
   プロ機能を"プリセット化してワンクリック"で出すのがByux流。フル機能エディタにはしない。
2. **既存資産（モーションブラー／BGMビート検出）が差別化の種**
   「速度リマップ×モーションブラー」「ダッキング×ビート検出」は、競合の単機能より"キル集特化"で勝てる組み合わせ。
3. **UIの簡単さは絶対防衛**
   AviUtl/Adobe/DaVince 共通の最大の弱点＝学習コスト。Byuxは「Kキー連打→自動分割」の手軽さが命。機能追加で複雑にせず、**プリセット/自動化で複雑さを隠す**。
4. **テンプレ/プリセット経済**
   Adobe(.mogrt)・DaVinci(PowerBins)・AviUtl(スクリプト)はみな"再利用資産"で伸びた。Byuxも「montageテンプレ共有」を持てば伸びしろ大。
5. **カラーグレードは費用対効果が最高**
   LUT一発で"プロっぽさ"。実装は軽い割にインパクト大。早めに入れる価値あり。

### ❌ 取り入れない（Byuxを殺す）
- 複雑なマルチトラック合成 / ノードUI / 汎用エディタ化 / 学習コストの高い無数の設定
- → これらはByuxの存在意義（速い・簡単・キル集特化）を壊す
